description: 기록된 내용을 바탕으로 task 작업 결과를 리뷰하고, 코드 구현 여부를 넘어 사용자의 기술적 역량 향상과 시야 확장에 초점을 맞춘 종합 피드백을 생성합니다.
---

이 워크플로우 실행 시 AI는 단순한 작업 완료 여부 확인을 넘어 **코드 리뷰어 및 멘토**의 역할을 수행해야 합니다:

1. `.agents/vault/records/` 디렉토리 하위에서 **최근에 생성된 작업 기록 파일**(예: `2026-03-02_auth-refactoring.md`)을 읽어들여 사용자의 초기 질문, 요청 사항, 태그(Tags), 그리고 기록된 학습 포인트 및 오답 노트(Troubleshooting Log)를 확인합니다.
2. 분석 전, 워크스페이스에 존재하는 기존 스킬(`/code-review` 또는 `/fix-issues`)의 논리(가벼운 문법 점검, Lint/Type-check 검토 등)를 백그라운드에서 결합하여 코드의 안정성과 실무적 품질 스캔을 함께 진행합니다.
3. 현재까지 수행된 작업 내역(코드 변경 사항, 구조 등)을 스캔 및 분석하여 최신 기록 파일에 명시된 각 요청 사항이 어떻게 처리되었는지 확인합니다.
4. 분석 결과를 바탕으로 **사용자의 기술적 성장**을 목적으로 하는 상세 피드백 리포트를 작성합니다. 피드백에는 다음이 포함되어야 합니다:
    - **핵심 기술 및 개념 요약**: 단순히 "코드를 이렇게 수정했다"가 아니라, 적용된 기술적 핵심 원리(Why)가 무엇인지, 어떤 문제를 어떤 방식으로 해결했는지 설명합니다.
    - **코드 품질 및 설계 리뷰**: 구현된 코드가 동작하더라도 더 나은 패턴(Best Practices), 안티 패턴(Anti-patterns) 여부, 유지보수성을 높이는 방법을 제안합니다.
    - **추가 학습 제안 (Next Steps)**: 사용자가 앞으로 더 깊이 공부해보면 좋을 관련 아키텍처적 고민, 디자인 패턴, 혹은 해당 스택/언어의 철학 등에 대한 키워드를 추천하여 시야 확장을 돕습니다.
    - **즉각적 실행 유도 (Actionable Next Step)**: 피드백을 단방향으로 끝내지 않고, 사용자가 방금 배운 개념이나 리팩토링 제안을 즉시 실험해 볼 수 있도록 **인터랙티브한 질문**을 한 가지 반드시 포함합니다. (예: *"제안해 드린 패턴을 지금 바로 코드에 적용해 보는 실습을 진행할까요?"*, *"추천한 키워드 중 하나에 대해 5분짜리 예제를 만들어 드릴까요?"*)
5. 작성된 멘토링 성격의 피드백 리포트를 마크다운 형식으로 정리하여 사용자에게 제공합니다.
