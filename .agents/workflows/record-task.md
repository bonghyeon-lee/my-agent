---
description: 현재 진행 중인 task에서 사용자의 질문과 요청을 기록하며, 단기적 해결보다는 사용자의 기술적 성장을 도모하기 위한 맥락과 학습 포인트를 파악합니다.
---

이 워크플로우를 실행하면 AI는 다음 단계를 수행해야 합니다:

1. 현재 대화 내역(Conversation History) 및 프롬프트를 분석하여 사용자가 요청한 주요 질문(Question)과 요구 사항(Task Request)을 파악합니다.
2. 단순히 해결해야 할 구현 목표뿐만 아니라, 사용자의 질문 속에 담긴 **기술적 고민, 오개념(Misconception), 낯설어하는 개념, 또는 더 알아두면 좋을 패턴**을 함께 식별합니다.
3. 작업 중 발생했던 에러 메시지나 막혔던 부분(Blockers)이 있다면, 이를 **오답 노트(Troubleshooting Log)** 항목으로 명시적으로 추출하여 실패 원인과 맥락을 함께 정리합니다.
4. 파악된 내용(요구 사항 + 식별된 학습 포인트 + 오답 노트)을 간결하고 명확하게 마크다운(Markdown) 형식으로 요약합니다.
5. 기존 단일 파일에 내용을 덧붙이는 방식 대신, 태스크마다 **새로운 개별 마크다운 파일**을 생성하여 기록합니다.
   - 파일명 규칙: `.agents/vault/records/{YYYY-MM-DD}_{주제어}.md` (예: `.agents/vault/records/2026-03-02_auth-refactoring.md`)
   - **단, 같은 날 같은 주제어의 파일이 이미 존재하는 경우 덮어쓰지 않고, 파일 하단에 새로운 타임스탬프와 함께 내용을 추가(Append)하여 기록합니다.**
   - 파일 최상단에는 **YAML Frontmatter**를 사용하여 태그(Tags) 메타데이터를 포함합니다. (예: `tags: [architecture, refactoring]`)
   - **[Domain MOC 연결]** 파일 본문 최상단(Frontmatter 직후)에는 현재 작업이 속한 도메인이나 대주제(예: `Domain: [[Store_Domain_MOC]]`, `Domain: [[UI_Components_MOC]]`)를 위키 링크로 명시하여 거시적인 그래프 군집(Cluster)을 형성합니다.
   - **[핵심 개념 노드화]** 본문 작성 시 과거 연관 작업뿐만 아니라, 새롭게 등장하거나 중요한 **핵심 기술 개념(Concept)**도 위키 링크(`[[React_Context_API]]`, `[[Hydration_Error]]` 등)로 감싸서, 향후 해당 개념을 허브(Hub)로 삼아 여러 문서가 연결될 수 있도록 지식 그래프(Graph)를 촘촘하게 구성합니다.
6. 어떤 이름의 파일이 생성되었고 어떤 태그와 학습 포인트가 기록되었는지 요약하여 사용자에게 알립니다.
