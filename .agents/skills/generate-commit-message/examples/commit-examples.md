# Commit Message Examples

## Good Examples

### New Feature

````text
feat: add social login support for Google and GitHub
```text

```text
feat: add usage breakdown tooltip to subscription status
```text

### Bug Fix

```text
fix: resolve memory leak in usage deduction service
```text

```text
fix: correct typo in environment variable key
```text

### Refactoring

```text
refactor: consolidate redundant modal components
```text

```text
refactor: apply VAC pattern to chat section structure
```text

### Chore

```text
chore: update dependencies and fix vulnerabilities
```text

---

## Bad Examples (To Avoid)

- `fix: bug` (너무 모호함)
- `Update file.js` (무엇이 업데이트되었는지 알 수 없음)
- `Modified some code to make it work better` (명확한 의도가 없음)
- `Reverted previous changes because it was wrong` (어떤 변경 사항을 되돌렸는지 명시 부족)
````
