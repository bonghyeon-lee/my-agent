# Code-Review Skill

이 스킬은 브랜치 머지 전 변경 사항을 분석하고 품질을 검토하는 데 도움을 줍니다.

## 주요 기능

1. **차이점 분석**: 현재 브랜치와 타겟 브랜치 간의 코드 차이점을 추출합니다.
2. **구조화된 리뷰**: 체크리스트를 바탕으로 코드의 기능성, 성능, 보안, 가독성을 검토합니다.
3. **자동화 검사**: 린트, 테스트 등의 자동화 도구 결과를 리뷰에 포함합니다.

## 사용 방법

1. **차이점 확인**:
   `scripts/get_branch_diff.sh [target-branch]` 스크립트를 사용하여 변경 사항을 확인합니다.
   (예: `./.agents/skills/code-review/scripts/get_branch_diff.sh yskim-hori/dev`)

2. **체크리스트 검토**:
   `resources/review_checklist.md`를 열고 변경 사항이 모든 항목을 충족하는지 확인합니다.

3. **피드백 작성**:
   리뷰 결과를 다음 형식으로 요약하여 보고합니다:
   - **변경 요약**: 어떤 기능이 추가/수정되었는가?
   - **주요 검토 사항**: 코드 리뷰 중 발견된 중요한 지점들
   - **수정 제안**: 개선이 필요한 코드 블록 및 이유
   - **검증 상태**: 테스트 및 빌드 성공 여부

## 리뷰 가이드라인

- **긍정적이고 건설적인 피드백**: "코드가 잘못되었습니다" 대신 "이 부분은 ~게 바꾸면 가독성이 더 좋아질 것 같습니다"와 같이 제안합니다.
- **컨텍스트 이해**: 변경의 목적을 먼저 파악하고, 그에 맞는 적절한 설계를 검토합니다.
- **품질 지향**: 당장의 동작뿐만 아니라 유지보수성과 확장성을 고려합니다.
