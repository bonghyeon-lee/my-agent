# Code Review Checklist

이 체크리스트는 머지 전 코드 품질을 유지하기 위해 확인해야 할 사항들입니다.

## 1. 전반적인 품질

- [ ] 코드가 가독성 있고 깔끔한가?
- [ ] 중복 코드가 없는가? (DRY 원칙)
- [ ] 변수 및 함수 이름이 명확하고 의미 있는가?
- [ ] 주석이 필요한 곳에 명확하게 작성되어 있는가? (복잡한 로직 등)

## 2. 기능성 및 로직

- [ ] 요구 사항을 모두 충족하는가?
- [ ] 엣지 케이스(Edge Cases)가 고려되었는가? (Null, 빈 값, 에러 상황 등)
- [ ] 디버깅을 위한 `console.log` 등이 제거되었는가?

## 3. 성능 및 보안

- [ ] 불필요한 반복문이나 무거운 연산이 없는가?
- [ ] 보안 취약점(SQL Injection, XSS 등)의 우려가 없는가?
- [ ] 민감한 정보(API Key 등)가 하드코딩되지 않았는가?

## 4. 테스트 및 빌드

- [ ] 새로운 기능에 대한 테스트 코드가 작성되었는가?
- [ ] 모든 테스트가 성공적으로 통과하는가? (`npm run test`)
- [ ] 빌드 오류가 없는가? (`npm run build`)
- [ ] 린트 오류가 없는가? (`npm run lint`)

## 5. UI/UX (해당하는 경우)

- [ ] 디자인 시안과 일치하는가?
- [ ] 반응형 레이아웃이 제대로 작동하는가?
- [ ] 접근성(Accessibility)이 고려되었는가?
