# Generate-PR-Message Skill

이 스킬은 PR을 올리기 전, 현재 브랜치의 커밋 목록과 변경된 파일들을 분석하여 표준화된 형식의 GitHub Pull Request 제목과 본문을 생성합니다.

## 주요 기능

1. **변경 사항 분석**: 대상 브랜치(기본: `yskim-hori/dev`)와 비교하여 커밋 목록 및 변경 파일을 수집합니다.
2. **PR 메시지 초안 생성**: 수집된 데이터를 바탕으로 PR 제목과 본문(개요, 변경 내용, 체크리스트)을 작성합니다.
3. **한국어 작성**: 팀 컨벤션에 맞게 PR 메시지를 한국어로 작성합니다.

## 사용 방법

1. **데이터 수집 스크립트 실행**:

   ```bash
   ./.agents/skills/generate-pr-message/scripts/collect_pr_data.sh [target-branch]
   ```

   생략 시 기본 브랜치(`yskim-hori/dev`)와 비교합니다.
   출력 결과(커밋 목록, 변경 파일 그룹)를 복사합니다.

2. **AI를 통한 PR 메시지 생성 요청**:
   스크립트 출력물을 붙여넣고 다음과 같이 요청합니다:

   > "위 내용을 바탕으로 PR 메시지를 작성해줘. `resources/pr_template.md`의 형식을 참고하되, 불필요하게 길어지지 않도록 꼭 필요한 **작업 목적**과 **핵심 변경 내용** 위주로 짧고 실용적으로 요약해 줘."

3. **검토 및 수정**:
   생성된 메시지를 검토하고, 필요한 경우 세부 내용을 보완합니다.

## PR 메시지 작성 가이드라인

- **제목**: `[타입] 한 줄로 핵심 변경 내용 요약` (예: `[feat] 구독 카드 UI 개선 및 사용량 실시간 반영`)
- **개요**: 이번 PR에서 무엇을, 왜 변경했는지 2~4문장으로 서술
- **변경 내용**: 기능/레이어별로 bullet point 형태로 구체적으로 나열
- **테스트**: 확인한 테스트 항목 및 방법 기술
- **참고 사항**: 리뷰어가 알아야 할 특이 사항, 후속 작업, 관련 이슈 등

## 타입 분류

| 타입       | 설명                            |
| ---------- | ------------------------------- |
| `feat`     | 새로운 기능 추가                |
| `fix`      | 버그 수정                       |
| `refactor` | 코드 구조 개선 (기능 변경 없음) |
| `chore`    | 빌드, 설정, 의존성 변경         |
| `docs`     | 문서 수정                       |
| `test`     | 테스트 추가/수정                |
| `style`    | UI/스타일 변경                  |
